Rate
Hover a rate to isolate it; click to pin, click again to release. Fast = single deploy 20 fps, or dual deploy drogue 100 fps + main 20 fps. Slow = single deploy 10 fps, or dual deploy drogue 80 fps + main 10 fps — all 4 numbers are the rate at this site's own ground level (a low-altitude test-drop number), not a fixed rate for the whole descent. The sim scales each faster for the thinner air higher up (same drogue/canopy falls faster in less dense air), so the actual drogue rate near a 30,000–50,000ft apogee can be 1.5–2.5x the ground number shown here, easing back down to it by the time the rocket's low.

Accuracy vs. final projection
Distance from each model's forecast point to the "final projection" point (HRRR's own post-launch analysis, not a forecast, run through this same descent sim -- see the History legend; not a real GPS landing, see "Real flight" if one's been recorded) for the currently-selected hour/deploy/rate/altitude. Format is total distance, then the (east, north) breakdown in feet. Color means the same thing across every launch, not just within this one table: a miss under 200ft is always green, and beyond that it's graded against how far the wind actually carried the rocket that day, so the same foot-error reads worse on a calm, short flight than a long, windy one.

Descent-only drift per model (GFS/HRRR/ECMWF/ICON/ARPEGE/GEM — only models with real pressure-level wind), integrated from apogee to ground, wind sampled every 50 ft of descent. Zone = convex hull of all models' points at that altitude, for both rate variants shown (single: 10/20 fps, dual: drogue 80/100 fps, main 10/20 fps) — all ground-level rates, scaled up per descent step for the thinner air at altitude (see the Rate legend's info button for how much). Point color = model, point shape = rate (circle = faster, square = slower) — hover any point for exact numbers. Not a go/no-go call — for a launch director to read against the actual approved landing zone once available.
